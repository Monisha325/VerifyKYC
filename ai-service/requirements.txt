# Face matching — dlib's ResNet face-encoder via the face_recognition wrapper.
# Replaces deepface/tensorflow/tf-keras/ArcFace: measured ~270MB peak RSS vs
# TensorFlow's 700MB+, which OOM-crashed Render's free tier (512MB).
#
# IMPORTANT: the official `dlib` PyPI package publishes NO prebuilt wheels at
# all (verified directly against PyPI's JSON API for 20.0.0/20.0.1 — sdist
# only), so `pip install dlib` always falls back to a from-source CMake build,
# which fails in this image (no cmake/build-essential/boost installed).
# `dlib-bin` is a community-maintained, binary-only fork of the same project
# that DOES publish real manylinux wheels (confirmed: cp312-manylinux_x86_64
# exists for 20.0.1, matching this Dockerfile's Python 3.12 base) and installs
# under the same `import dlib` name, so face_recognition works unmodified.
#
# face_recognition itself declares a dependency on the package named `dlib`
# (not `dlib-bin`), so it's installed separately in the Dockerfile with
# --no-deps — its other real dependencies (face-recognition-models, click,
# numpy, pillow) are still listed normally below and installed by this file.
